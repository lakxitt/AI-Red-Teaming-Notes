# RAG and Vector Databases

RAG improves AI responses by retrieving relevant information from an external knowledge source before generating an answer.

Architecture:
User Query -> Embedding Model -> Vector Database -> Retrieve Relevant Documents -> LLM -> Final Response.

Embeddings are numerical vector representations of text.
Example: [0.21, -0.84, 1.22, ...]

Vector databases search by semantic meaning rather than keywords.
Popular databases: FAISS, ChromaDB, Pinecone, Milvus.
